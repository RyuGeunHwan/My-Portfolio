학원 게시판 사이트 제작


수업시간에 진행했던, CRUD를 기초로 간단한 학원 게시판을 개발했습니다.
개발기간은 약 2주 정도 소요되었고 처음으로 클라우드에 배포까지 해보았습니다.

* 프로젝트를 진행한 이유?
이유 : 기본적인 SPRING의 개념들을 최대한 이해하기 위해서 게시판이 최적화 되어있다 판단하여 프로젝트를 진행하게 되었습니다.
* 느낀점과 보완점
1. RestController와 Controller를 따로 만들어 백엔드와 프론트가 통신하는 그림을 공부하게 되었습니다.
2. AJAX를 이용하여 비동기처리하는 방식을 알게됨과 동시에 AJAX의 장단점을 알게되었습니다.
3. MVC패턴을 베이스로 하여 Controller, Service, Mapper 구조를 공부하는데 중점을 두었습니다.
4. 간단한 log interceptor를 통해 interceptor의 진행 구조와 개념을 알게 되었습니다.
5. spring security와 session을 통하여 회원가입시 비밀번호 암호화, session을 통하여
DB의 데이터와 로그인 시 데이터를 비교하여 로그인 진행 여부 확인을 하였습니다.
6.update, delete, insert는 VO 객체로 매핑하고, select는 HashMap을 매핑하는 규칙도 정해 보았습니다.
7. WAR와 JAR의 차이점을 알고 AWS EC2에 WAR파일을 최종 배포(deploy) 했습니다.
8. ORM을 이용하여 관계형 DB의 데이터를 자동 매핑을 하였습니다.
9. Spring의 특징인 IoC(제어의역전), DI(의존성 주입)를 통하여 조금 더 개선된 코드를 사용하기 위해 노력했습니다.
10. Students 데이터의 보안을 조금 더 강화하고 게시판의 수정과 삭제 등 권한 컬럼을 보완해야 완성에 가까운 게시판이 될 것 같습니다.

해당 게시판은 board 여기서 확인이 가능합니다. 아이디는 "비밀번호777"이고 비밀번호는 "777"입니다.
